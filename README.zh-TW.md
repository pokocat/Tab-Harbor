# TabHarbor

TabHarbor 是一款為「分頁開太多，但又不想粗暴全部關掉」的使用者設計的 Chrome 擴充功能。

## 設計哲學

TabHarbor 的核心理念很簡單：分頁管理應該降低認知負擔，而不是再增加一套新的整理成本。

- 以工作上下文為中心：分頁通常屬於某個暫時性的工作情境，而不是長期收藏。
- 清理必須可逆：封存、去重、關閉都應該讓使用者安心，因為之後容易找回。
- 先快速看清現況：介面應該幫助使用者快速理解「現在到底開了哪些分頁」。
- 輕結構而非強約束：分組、篩選與建議是輔助，而不是強迫改變使用習慣。
- 預設本地儲存：工作階段與設定保存在瀏覽器本地，權限盡量保持最小。

## 主要功能

- 在 popup 中快速檢視目前視窗的分頁，並執行常用操作。
- 自動偵測重複分頁，支援一鍵清理。
- 支援工作階段封存與還原，方便暫時收起目前的工作上下文。
- 提供高資訊密度的 dashboard，用於跨視窗、跨網域整理分頁。
- 支援搜尋開啟中的分頁與已封存工作階段。
- 支援依根網域分組，以及依時間或標題排序。
- 支援按鈕音效開關。
- 支援多語系介面，預設跟隨瀏覽器語言，也可在 dashboard 手動切換。

## 互動結構

TabHarbor 採用兩層使用方式：

- Popup：適合快速檢視、清理目前視窗、去重與快捷搜尋。
- Dashboard：適合深度整理、大量操作、工作階段管理、篩選與排序。

這樣既保留高頻操作的輕量感，也能在分頁很多時提供完整的管理空間。

## 資料與隱私

- 工作階段與偏好設定使用 `chrome.storage.local` 本地保存。
- 分頁管理只使用核心功能所需的最小權限。
- 核心體驗不依賴遠端同步或分析服務。
