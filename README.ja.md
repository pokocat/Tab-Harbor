# TabHarbor

TabHarbor は、「タブを開きすぎているけれど、全部を乱暴に閉じたくはない」というユーザーのための Chrome 拡張機能です。

## 設計哲学

TabHarbor の中心にある考え方はシンプルです。タブ管理は、新しい作業負担を増やすのではなく、認知負荷を減らすべきだということです。

- 文脈を優先する: タブは長期保存用のブックマークではなく、一時的な作業コンテキストの一部であることが多いです。
- クリーンアップは元に戻せるべき: アーカイブ、重複整理、クローズを安心して行えるように、復元しやすさを重視します。
- まず現状を素早く把握する: 「今何が開いているのか」を短時間で理解できる UI を目指します。
- 厳格な整理ではなく、やさしい構造化: グループ化、検索、提案は支援であり、ワークフローの強制ではありません。
- ローカルファースト: セッションと設定はブラウザ内に保存され、権限は最小限に抑えます。

## 主な機能

- Popup で現在のウィンドウ内のタブをすばやく確認し、主要操作を実行。
- 重複タブを自動検出し、ワンクリックで整理。
- セッションのアーカイブと復元により、作業コンテキストを安全に一時退避。
- 高密度な dashboard で、ウィンドウ横断・ドメイン横断の整理を実現。
- 開いているタブとアーカイブ済みセッションの検索。
- ルートドメインごとのグループ化と、時間・タイトルによる並び替え。
- ボタン音のオン・オフ切り替え。
- 多言語 UI をサポートし、既定ではブラウザ言語に追従。dashboard から手動変更も可能。

## インターフェース構成

TabHarbor は 2 層の操作モデルを採用しています。

- Popup: 現在のウィンドウの確認、軽い整理、重複削除、クイック検索向け。
- Dashboard: 大量のタブ整理、セッション管理、フィルタ、ソート、まとめて操作するための画面。

この構成により、日常的な操作は軽く、タブ数が多いときはより深い整理ができるようになっています。

## データとプライバシー

- セッションと設定は `chrome.storage.local` に保存されます。
- タブ管理にはコア機能に必要な最小限の権限のみを使用します。
- 基本体験にリモート同期や解析サービスは不要です。
