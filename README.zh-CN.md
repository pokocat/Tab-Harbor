# TabHarbor

TabHarbor 是一个面向“标签页开太多，但又不想粗暴全部关掉”的用户设计的 Chrome 插件。

## 设计哲学

TabHarbor 的核心想法很简单：标签管理应该降低认知负担，而不是再制造一套额外的整理成本。

- 以上下文为中心：标签页往往属于某个临时工作上下文，而不是长期书签。
- 清理必须可逆：归档、去重、关闭都应该让用户有安全感，因为之后容易找回。
- 尽快看清现状：界面应该帮助用户快速理解“现在到底开了什么”。
- 轻结构而非强约束：分组、筛选和建议是辅助，而不是强迫用户改变习惯。
- 默认本地化：会话和设置保存在浏览器本地，权限尽量最小化。

## 主要功能

- 在 popup 中快速查看当前窗口标签，并执行常用操作。
- 自动识别重复标签，并支持一键清理。
- 支持会话归档与恢复，便于临时收起当前工作上下文。
- 提供高信息密度的 dashboard，用于跨窗口、跨域名整理标签。
- 支持搜索打开中的标签和已归档会话。
- 支持按根域名分组、按时间或标题排序。
- 支持按钮声音开关。
- 支持多语言界面，默认跟随浏览器语言，也可在 dashboard 中手动切换。

## 交互结构

TabHarbor 分为两层使用方式：

- Popup：适合快速查看、清理当前窗口、去重和快捷搜索。
- Dashboard：适合深度整理、大批量操作、会话管理、筛选与排序。

这样既保留了高频操作的轻量感，也能在标签很多时提供更完整的管理空间。

## 数据与隐私

- 会话和偏好设置使用 `chrome.storage.local` 本地保存。
- 标签管理只使用实现核心功能所需的最小权限。
- 核心体验不依赖远程同步或分析服务。
